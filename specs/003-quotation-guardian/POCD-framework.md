## Opportunity（機會）

### 市場規模估算（台灣室內設計產業）

| 層級 | 描述 | 估算 |
|------|------|------|
| TAM | 台灣所有室內設計公司（~15,000+ 家） | — |
| SAM | 有 3+ 名設計師、年接 20+ 案的中型設計公司 | ~2,000-3,000 家 |
| SOM | 第一年能觸及的（Vivian 人脈 + 口碑擴散） | ~50-100 家 |

### 為什麼是現在

1. **AI 成本崩塌** — Gemini 2.5 Flash 讓 PDF 解析從「不可能」變成「幾乎免費」
2. **裝修市場持續成長** — 台灣老屋翻新需求不斷上升，設計公司產能瓶頸加劇
3. **數位化斷層** — 室內設計公司仍以 Excel 報價為主，市場上沒有「風險管理」角度的工具
4. **經驗斷層危機** — 資深設計師退休潮，年輕設計師缺乏經驗傳承管道

### 競爭格局

| 類型 | 代表 | Q-Check 的差異 |
|------|------|---------------|
| 傳統報價軟體 | 各種 Excel 範本 | 沒有風險引擎、沒有知識傳承 |
| 裝修平台 | 100室內設計、設計家 | 面向消費者，非面向設計公司 |
| 工程管理 SaaS | 明基逐鹿、鼎新 | 太重、太貴，專注施工管理而非報價階段 |
| AI 報價工具 | 尚未出現 | 多數專注「自動生成」，Q-Check 專注「風險攔截」|

**核心差異化：沒有人從「風險管理」角度切入報價流程。**

## Context（環境脈絡）

### 有利因素

- 台灣裝修糾紛頻繁（消保會每年大量申訴），設計公司有動機降低風險
- 年輕設計師教育成本高，老闆願意投資「讓年輕人少犯錯」的工具
- Cloudflare Pages + Workers 部署成本趨近零，能在 revenue 前長期運營

### 不利因素

- 室內設計產業數位化程度低，導入新工具的阻力大
- 每間公司報價格式、工項分類、管理文化都不同，標準化困難
- 設計師工作節奏忙碌，學習新系統的意願低
- 台灣 SaaS 付費意願普遍偏低（尤其傳統產業）

### 關鍵假設（需驗證）

| # | 假設 | 驗證方式 | 狀態 |
|---|------|---------|------|
| 1 | 設計師願意在系統中開立報價（而非只上傳 PDF） | Vivian 實際使用 | 待驗 |
| 2 | 「含/不含」欄位能有效減少認知落差 | 追蹤追加金額變化 | 待驗 |
| 3 | 資深覆核時間確實能減少 70% | 實測覆核報告流程 | 待驗 |
| 4 | 其他設計公司的痛點與 Vivian 公司類似 | 3-5 家 beta 用戶訪談 | 待驗 |

## Deal（商業模式與時程）

### 商業模式演進

```
Phase 1 (0-6月)：內部工具 → 免費
  ↓ 量化成效：追加金額 ↓、覆核時間 ↓、修改次數 ↓
Phase 2 (6-18月)：SaaS 訂閱 → NT$ 3,000-8,000/月（按公司人數）
  ↓ 有付費客戶 + 數據累積
Phase 3 (18月+)：產業作業系統 → 增值服務（行情數據、工班媒合）
```

### 單位經濟（Phase 2 目標）

| 指標 | 目標 |
|------|------|
| ARPU | NT$ 5,000/月 |
| 目標客戶數（Year 1） | 50 家 |
| MRR 目標 | NT$ 250,000 |
| CAC | 口碑 + Vivian 社群 → 趨近 0 |
| LTV/CAC | 極高（因 CAC ≈ 0） |

**資金需求：** 目前不需要外部資金。技術開發 Stephen 獨立完成，基礎建設成本（Cloudflare）趨近零。
